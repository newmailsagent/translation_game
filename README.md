# Screen Link

Трансляция экрана, звука игры и микрофона одному другу в высоком качестве — без сервера, без VPS и без домена. Один HTML-файл, размещается на **GitHub Pages**.

- Видео до 4K@60, битрейт до 50 Мбит/с, звук Opus stereo 510 кбит/с
- Микрофон для комментариев: смешивается со звуком игры, кнопка mute во время трансляции
- Вся передача — напрямую между вами и другом (WebRTC P2P, шифруется DTLS-SRTP)
- Один длинный секретный токен вместо аккаунтов
- Клиентский rate-limit: страница хоста принимает не больше 5 попыток подключения в минуту

## Как это работает без сервера

GitHub Pages раздаёт статическую страницу. Для «знакомства» двух браузеров используется бесплатное публичное облако PeerJS — через него проходит только служебная информация об установке связи, само видео и звук идут напрямую между компьютерами.

Ваш токен в облако PeerJS не отправляется: имя комнаты — это SHA-256-хэш токена, а зритель дополнительно предъявляет токен целиком по зашифрованному каналу, и хост сверяет его перед тем, как отдать поток.

## Размещение (один раз, 2 минуты)

1. Создайте репозиторий на GitHub и положите в него `index.html`.
2. Settings → Pages → Source: `main`, папка `/ (root)` → Save.
3. Страница появится на `https://ваш-ник.github.io/имя-репозитория/`.

## Использование

1. Придумайте токен (мин. 16 символов, лучше сгенерировать):
   в консоли браузера `crypto.randomUUID() + crypto.randomUUID()`.
2. Вы открываете `https://ваш-ник.github.io/репо/#ВАШ_ТОКЕН`, при желании оставляете галочку микрофона → «Транслировать мой экран».
3. В диалоге выбора экрана включите **«Поделиться звуком»** (галочка внизу в Chrome). Полноценный звук всего экрана захватывается в Chrome/Edge на Windows; на macOS системный звук доступен при захвате вкладки браузера.
4. Друг открывает ту же ссылку → «Смотреть трансляцию».

Токен стоит передавать другу лично (мессенджером с шифрованием), а не публиковать. Часть ссылки после `#` не отправляется по сети при загрузке страницы.

## Ограничения — честно

- **Сигналинг зависит от облака PeerJS.** Оно бесплатное и обычно стабильное, но это чужой сервис: если он ляжет, новые подключения не установятся (уже идущая трансляция не пострадает — она серверов не использует).
- **Серверного rate-limit нет** — его негде разместить без своего сервера. Вместо него: комнату невозможно найти без токена (43+ случайных символа перебором не подбираются), хост сверяет токен и игнорирует шквал попыток на своей стороне.
- **Строгий NAT.** Если у обоих провайдеров симметричный NAT, прямое P2P-соединение может не установиться — понадобится TURN-сервер, а это уже чей-то сервер. На домашних тарифах такое встречается редко.
- **Один зритель.** Ровно хост и один друг; новый зритель вытесняет предыдущего.
- **Качество упирается в ваш исходящий канал.** Для 1440p60 комфортно от 20–30 Мбит/с аплоада.
